# 更新日志

## v0.4.4 - 2026-07-18

- 修复部分 AstrBot 版本或 Provider 返回结构下，`/良语` 未知缩写无法调用模型推断的问题。
- 模型响应文本提取兼容 `result_chain`、`completion_text`、字符串、dict 和常见 OpenAI `choices` 结构。
- 当模型推断仍失败时，回复中会附带短错误原因，便于继续排查。

## v0.4.3 - 2026-06-29

- 静默理解不再改写用户消息文本，改为在 LLM 请求阶段注入系统提示。
- 移除可能被模型复述的 `[良语理解提示：仅供理解原消息，不要复述本段]` 文本。

## v0.4.2 - 2026-06-29

- 群消息自动识别收紧为仅识别带 `·`、`•`、`・` 等良语专用分隔符的缩写。
- 不再把逗号、句号、空格或无分隔符文本当作良语，修复 `好了，洗掉了翻译说明` 被误识别的问题。
- 移除已无意义的无分隔符匹配配置说明。

## v0.4.1 - 2026-06-29

- 修复模型推断结果可能带出“翻译说明”“可以理解为”等说明性文本的问题。
- 调整静默理解上下文措辞，避免主模型复述插件内部提示。

## v0.4.0 - 2026-06-29

- 普通对话中出现良语时，默认改为静默理解，不再主动输出翻译。
- 当用户使用 `/良语` 或明确要求翻译、解释、询问含义时，才主动回复翻译结果。
- 静默理解时会把良语释义追加给主模型，帮助 bot 理解输入语义。
- 新增 `auto_reply_on_explicit_request`、`silent_understand_normal_messages`、`silent_infer_unknown` 配置项。

## v0.3.0 - 2026-06-29

- 未知良语推断时接入当前会话人格设定。
- 未知良语推断时尝试检索 AstrBot 知识库/数据库，并把相关剧情上下文交给模型。
- 新增 `use_persona_context`、`use_knowledge_base`、`knowledge_context_max_chars` 配置项。

## v0.2.0 - 2026-06-29

- 词库未命中时调用 AstrBot 当前 LLM，根据良语例句规律推断未知缩写。
- 新增未知带分隔符良语候选提取、推断提示词构造和模型输出清理。

## v0.1.0 - 2026-06-29

- 初始版本。
- 从 `良·语举例.xlsx` 提取内置良语词库。
- 支持群消息良语识别、`/良语` 查询命令和基础配置。
